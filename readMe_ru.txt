Пример интеграции SAP GUIx32 для Windows с Приложением java8x64 через мост ActiveX.

Для того что бы запустить пример необходимо зарегистрировать Activex BridgeX4Java.dll ./CPP/bin/
BridgeX4Java.dll - 32x разрядная библиотека для обмена сообщения с Java сервером.
В данном примере обмен сообщениями происходит через сокет.
Сервер Java отдельный x64 процесс, что существенно расширяет доступную для использования память.

В качестве примера вызова с параметром реализован метод TESTEXCHANGE, на java сервере 
String testExChange(String testString) в интерфейсе IGridStub.

Можно расширить набор методов Java доступных для вызова из abap. Пример такого расширения описан в newСallSample.txt.
Для того что бы расширить набор необходимо добавить в интерфейс описание метода 
void Sample_With_3_Args(String string_Arg_1, int int_Arg_1, String string_Arg_2);
И сделать его имплементацию в ServerJGridBean01.

Как можно видеть из примера общий формат описания вызова: "Method_name;Method_Id;Param_1;Param_2;...;Param_n"
Естественно возникает вопрос, что если строковый параметр содержит в себе разделитель ';' в этом случае 
можно использовать какую либо параметра, например Url Ecoding,a на стороне сервера Url Decoding.

Так же имеется возможность указать серверу что вызов метода необходимо осуществить в отдельном потоке, 
(это иногда  необходимо что бы не блокировать диалоговый обмен SAP GUI и SAP BackEnd) для этого необходимо 
передать параметр @@THR в массиве параметров: "Method_name;Method_Id;@@THR;Param_1;Param_2;...;Param_n".
см. так же в newСallSample.txt
